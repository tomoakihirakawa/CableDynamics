# 橋梁ケーブルへの風荷重モデル — ランプドマス枠組みでの段階実装

## 0. 本書の位置づけ

[memo.md](memo.md) で整理した「橋梁ケーブル風応答解析への拡張」のうち，
**風荷重の与え方**だけを切り出して詳細化したもの．以下の制約条件を守る：

- **既存ランプドマス枠組み**を保つ．`networkPoint` に集中質量を持たせ，
  `networkLine` から張力・抗力・揚力を集約する設計を変えない．
- **`MooringLine::simulate()` のサブステップ機構**（[MooringLine.hpp:131-205](../lib/include/MooringLine.hpp#L131-L205)）
  をそのまま使う．BEM 側 Δt の中で CFL 内部刻みでサブ積分する構造を維持．
- 新しい流体ソルバを書かない．**経験的・準定常的な力モデル**で完結させる．

簡単なものから順に Level 0 → Level 4 まで5段階で構成する．
各 Level は前の Level の上に**独立して上乗せできる**設計にし，
任意の段階で打ち切れるようにする．

## 1. 物理現象の整理

橋梁ケーブルの風応答で扱われる主要現象は以下の4つ．それぞれ励起機構・必要モデル・
支配パラメータが異なるので，どこまで再現したいかを最初に決めるのが重要．

| 現象 | 励起機構 | 風速範囲 | 振動数 | 必要モデル | 本書 Level |
|---|---|---|---|---|---|
| **静的たわみ** | 平均風による定常抗力 | 全域 | 0 (DC) | 定常 drag | **L0** |
| **バフェッティング** | 風乱流の広帯域加振 | 中〜高 | 多モード励振 | 準定常＋乱流入力 | **L3** |
| **VIV（渦励振）** | 渦放出（Strouhal） | 低〜中 | 単一モード支配 | 強制揚力 or wake oscillator | **L1, L4** |
| **ギャロッピング** | 負の空力減衰（Den Hartog） | 高 | 構造1次モード | 準定常＋$C_L(\alpha)$ | **L2** |
| **レインバイブレーション** | 水滴ライン形成＋負減衰 | 中 | 構造1次モード | 専用モデル | スコープ外 |

実橋ケーブルの定常的な振動問題は **VIV とレインバイブレーション**が支配的だが，
台風等の極値設計では**バフェッティングとギャロッピング**が効く．

### 共通の出発点：準定常理論（quasi-steady theory）

時間領域の橋梁振動解析では，瞬間的な相対風速 $\vec U_\text{rel}$ に対して
定常空力係数 $C_D, C_L$ を当てはめる**準定常理論**が業界標準である．
非定常流体メモリ効果（Theodorsen 関数等）は無視するが，**振動振幅評価には十分**で，
長大橋・ケーブルの設計実務でも採用されている．[¹][²]

## 2. 共通の数学的下地

ここで導入する記号は以降すべての Level で共通．

### 2.1 ケーブル局所幾何

`networkPoint` $i$ にぶら下がる `networkLine` $l$ について，

- $\hat t_l$: 線分の単位接線ベクトル
- $D$: ケーブル直径（`networkLine::diameter`）
- $L_0$: 線分の自然長（`networkLine::natural_length`）
- $\Delta s_l = $ 線分の現長

### 2.2 相対風速とその法線成分

風速場を $\vec U_\text{wind}(\vec X, t)$，節点速度を $\vec v_i$ とする．
**相対風速**

$$\vec U_\text{rel} = \vec U_\text{wind}(\vec X_i, t) - \vec v_i$$

円柱の**独立性原理**（independence principle）に従い，軸方向成分は空力に
寄与しないとして除く．**法線成分**は

$$\vec U_n = \vec U_\text{rel} - (\vec U_\text{rel}\cdot\hat t_l)\,\hat t_l,
\qquad U_n \equiv |\vec U_n|$$

**揚力方向単位ベクトル**は接線と $\vec U_n$ の双方に直交する：

$$\hat n_L = \frac{\hat t_l \times \vec U_n}{|\hat t_l \times \vec U_n|}$$

### 2.3 ケーブル単位長さあたりの空力

最も一般的な準定常空力（抗力＋揚力）は

$$\vec f_\text{aero}(s,t)
= \tfrac{1}{2}\rho_\text{air}\,D\,U_n
\Bigl[\,C_D\,\vec U_n + C_L\,U_n\,\hat n_L\,\Bigr]$$

ここで $\rho_\text{air} = 1.225\,\text{kg/m}^3$ は空気密度．
これを線分長 $\Delta s_l$ で積分し，両端節点に半分ずつランプする：

$$\vec F_i \mathrel{+}= \tfrac{1}{2}\,\vec f_\text{aero}(s_l,t)\,\Delta s_l$$

これが**ランプドマス流儀の集約**で，既存の `getDragForce()` の構造と完全に同じ．

### 2.4 重要な無次元量

設計実務でよく使われる無次元数．本書の検証ケースで参照する．

| 名称 | 定義 | 意味 |
|---|---|---|
| **Reynolds 数** | $Re = U_n D/\nu_\text{air}$ | 流れの慣性／粘性比．$\nu_\text{air}\approx 1.5\times 10^{-5}$ m²/s |
| **Strouhal 数** | $St = f_s D/U_n$ | 渦放出周波数の無次元化．円柱で $St\approx 0.20$（亜臨界域）[³] |
| **換算流速** | $U_r = U/(f_n D)$ | 構造振動数で正規化．VIV ロックインは $U_r\approx 5$ |
| **Scruton 数** | $Sc = 2\mu\zeta_s/(\rho_\text{air} D^2)$ | VIV 振幅の支配パラメータ．$Sc\gtrsim 10$ で抑制 |
| **質量比** | $m^* = \mu/(\tfrac{\pi}{4}\rho_\text{air}D^2)$ | 構造質量と排除流体質量の比 |

ここで $\mu$ はケーブル線密度 [kg/m]，$\zeta_s$ は構造減衰比．

## 3. 段階実装プラン

### Level 0 — 定常抗力のみ（既存 `getDragForce` 改修）

**目的**：水中→空気の最小切替．静的たわみを再現する．

#### 物理

時間依存・節点速度依存をすべて落とした定常抗力：

$$\vec f_\text{aero} = \tfrac{1}{2}\rho_\text{air}\,D\,U_\text{wind}^2\,C_D\,\hat U_\text{wind}$$

#### 必要な実装

**[Network.hpp:609-636](../lib/include/Network.hpp#L609-L636)** の `getDragForce()` を雛形に，
新しいメンバ関数を追加する：

```cpp
std::array<double,3> getAeroForce(
   const std::function<Tddd(Tddd, double)>& wind_field,
   double current_time,
   double rho_air = 1.225) const
{
   std::array<double,3> aero_force;
   aero_force.fill(0);
   for (const auto& l : this->getLines()) {
      auto* other = (*l)(this);
      // セグメント幾何
      auto A2B = other->RK_X_sub.getX() - this->RK_X_sub.getX();
      double seg_len = Norm(A2B);
      if (seg_len < 1e-12) continue;
      auto t_hat = A2B / seg_len;

      // 相対風速 → 法線成分
      auto U_wind = wind_field(this->RK_X_sub.getX(), current_time);
      auto U_rel = U_wind - this->RK_velocity_sub.getX();
      auto U_n_vec = U_rel - Dot(U_rel, t_hat) * t_hat;
      double U_n = Norm(U_n_vec);
      if (U_n < 1e-12) continue;

      // 抗力（半セグメント分）
      double Cd = l->Cd;             // 新メンバ，後述
      double half_area = 0.5 * l->diameter * seg_len;
      aero_force += 0.5 * rho_air * U_n * Cd * half_area * U_n_vec;
   }
   return aero_force;
}
```

#### 必要な追加メンバ

`networkLine` ([Network.hpp:362-366](../lib/include/Network.hpp#L362-L366)) に：

```cpp

double Cl0     = 0.0;  //! 平均揚力係数

double St      = 0.20; //! Strouhal 数
double Cl_rms  = 0.0;  //! VIV揚力 RMS（Level 1で使用）
```

既存の海中向け `getForce()` は壊さず，**橋梁用の派生クラス**を作るのがクリーン：

`MooringLine::simulate()` の力評価ループ
（[MooringLine.hpp:166-172](../lib/include/MooringLine.hpp#L166-L172)）を派生で
オーバーライドできるよう，将来的に仮想化を検討．

#### CFL

Level 0 はプリテンションが支配的なので，[memo.md](memo.md) の CFL 議論
$\Delta t \lesssim L_0/\sqrt{T_0/\mu}$ がそのまま使える．既存サブステップ機構で十分．

#### 検証

1. **無風で自由振動**：弦理論 $f_n = (n/2L)\sqrt{T_0/\mu}$ を再現するか
2. **一様風で静的たわみ**：両端ピン支持の弦に分布荷重 $w = \tfrac{1}{2}\rho_\text{air}DU^2 C_D$
   を作用させたときの最大たわみ $\delta_\text{max} = wL^2/(8T_0)$ を再現するか
3. **空力減衰**：自由振動させて減衰増分を測り，$\zeta_\text{aero} \approx \rho_\text{air}UDC_D/(2\mu\omega_n)$ と一致するか

### Level 1 — 強制揚力モデル（forced excitation）

**目的**：単一周波数の揚力で VIV の弱いロックイン外応答とギャロッピング非有り
応答（線形領域）を再現．

#### 物理

セグメント中心周波数を Strouhal 則で決め，正弦的な揚力を加える：

$$\vec f_\text{lift}(s,t)
= \tfrac{1}{2}\rho_\text{air}\,D\,U_n^2\,\widetilde{C}_L\,\Psi(s,t)\,\hat n_L$$

$\widetilde{C}_L$ は実効揚力係数（円柱で $\sim 0.3$）[³]，$\Psi$ は無次元時間関数：

$$\Psi(s,t) = \sin\bigl(\varphi_l(t) + \phi_l\bigr)$$

ここで $\varphi_l(t)$ は **位相を時間積分形で更新**することで $f_s$ の時間変動に
対応する：

$$\varphi_l(t+\Delta t) = \varphi_l(t) + 2\pi f_s(s_l,t)\,\Delta t,
\qquad f_s = St\cdot U_n/D$$

$\phi_l$ は**初期ランダム位相**で，**軸方向相関長 $L_c \approx 5D$ ごとに同じ値を割当**．
これがないとケーブル全体が同位相で振動し，第1モードだけ過剰励振される非物理が出る．

#### 必要な追加メンバ

`networkLine` に：

```cpp
double vortex_phase = 0.0;     //! φ_l(t)：時間積分される位相
double vortex_phase0 = 0.0;    //! φ_l(0)：初期位相（相関ブロックで共有）
int    correlation_block = -1; //! どの空間相関ブロックに属するか
```

#### 位相の初期化

セグメント数を $N_l$，相関ブロック数を $N_b = \max(1, \lfloor L_\text{cable}/(5D)\rfloor)$ とする．
線形 cable の場合は等分割：

```cpp
void initializeVortexPhases(int n_blocks, std::mt19937& rng) {
   std::uniform_real_distribution<double> uni(0., 2*M_PI);
   std::vector<double> block_phase(n_blocks);
   for (auto& p : block_phase) p = uni(rng);

   auto lines = ToVector(this->getLines());
   int N = lines.size();
   for (int i = 0; i < N; ++i) {
      int b = (i * n_blocks) / N;     // 等分配
      lines[i]->correlation_block = b;
      lines[i]->vortex_phase0 = block_phase[b];
      lines[i]->vortex_phase  = block_phase[b];
   }
}
```

#### 揚力を含む力計算

`getAeroForce()` に揚力項を追加：

```cpp
// 抗力に加えて
double Cl_amp = l->Cl_rms * std::sqrt(2.0);  // RMS → 振幅変換
double psi = std::sin(l->vortex_phase);      // 時間項
auto n_L = Cross(t_hat, U_n_vec) / Norm(Cross(t_hat, U_n_vec));
aero_force += 0.5 * rho_air * U_n * U_n * l->diameter * 0.5 * seg_len
              * Cl_amp * psi * n_L;
```

#### 検証

1. **励起周波数のスイープ**：風速 $U$ をゆっくり上げて，振動振幅が
   $f_s = f_n$（つまり $U \approx f_n D/St$）でピークを示すか
2. **モード番号の確認**：軸方向相関長 $L_c$ を変えて，励起されるモード番号が
   $n \approx L/(2L_c)$ 付近にシフトするか
3. **線形応答理論との比較**：定常 RMS 振幅
   $$\sigma_u^{(n)} = \frac{F_\text{rms}^{(n)}}{2\,M_n\,\omega_n^2\,(\zeta_s+\zeta_\text{aero})}$$
   と一致するか（$M_n$ はモード質量，$F_\text{rms}^{(n)}$ はモードへの励起力 RMS）

#### Level 1 の限界

- **ロックインしない**：渦周波数は構造振動数に引き寄せられない．
  ロックインを再現するには Level 4（wake oscillator）が必要．
- **単色で帯域がない**：実際の渦は帯域を持つ．Level 2 以降で改善．
- **空間相関がブロック定数**：滑らかな空間コヒーレンスにはなっていない．Level 3 で改善．

### Level 2 — 帯域広がり付き揚力＋準定常 $C_L(\alpha)$

**目的**：(a) 渦放出周波数の自然な広がりを表現，
(b) 迎角依存の空力係数によりギャロッピング不安定を再現．

#### (a) 周波数バンド

渦放出周波数 $f_s$ は単色ではなく，**ガウシアンバンド**を持つ：

$$S_F(f) = F_0^2\,\frac{1}{\sqrt{2\pi}\sigma_f}
\exp\!\left(-\frac{(f-f_s)^2}{2\sigma_f^2}\right)$$

経験的に $\sigma_f/f_s \in [0.05, 0.15]$．時系列への変換は
**離散コサイン級数**（Shinozuka 法[⁴]の最簡版）：

$$\Psi(s,t) = \sum_{k=1}^{N_f} a_k\cos\bigl(2\pi f_k\,t + \phi_l + \theta_k\bigr)$$

- $f_k = f_s - 3\sigma_f + (k-\tfrac{1}{2})\Delta f$，$\Delta f = 6\sigma_f/N_f$
- $a_k = \sqrt{2\,S_F(f_k)\,\Delta f}/F_0$（規格化済み）
- $\theta_k$：各成分に独立な一様乱数位相
- $\phi_l$：セグメント／ブロック固有の初期位相（Level 1 と同じ）

$N_f \approx 10\sim 30$ で十分．$\sum_k a_k^2 = 1$ になるように規格化すれば，
$\widetilde{C}_L$ が直接 RMS 揚力係数になる．

#### 必要な追加メンバ

```cpp
struct VortexBand {
   std::vector<double> a_k;       // 振幅係数（規格化済み）
   std::vector<double> theta_k;   // 周波数成分位相
   std::vector<double> df_k;      // f_k - f_s（中心からのずれ，無次元）
};

// networkLine に：
VortexBand vortex_band;
double sigma_f_ratio = 0.1;  // σ_f / f_s
int    n_freq        = 16;   // N_f
```

#### 時間項の評価

```cpp
double psi = 0.0;
double total_phase_t = current_time;
for (size_t k = 0; k < l->vortex_band.a_k.size(); ++k) {
   double fk = (1.0 + l->vortex_band.df_k[k]) * fs;  // 中心 fs に対する係数
   psi += l->vortex_band.a_k[k] *
          std::cos(2*M_PI*fk*total_phase_t + l->vortex_phase0 + l->vortex_band.theta_k[k]);
}
```

注：ここでは位相を時間積分せずに $\cos(2\pi f_k t)$ で書いた．$f_s$ が時間変動する
場合は Level 1 と同様に $\varphi_k(t+\Delta t) = \varphi_k(t) + 2\pi f_k\Delta t$ で
更新する形に書き直す．

#### (b) 迎角依存の $C_L, C_D$

矩形断面・H 型・D 型などの非円形ケーブルでは $C_D, C_L$ が迎角 $\alpha$ に依存する．
迎角は

$$\alpha = \alpha_0 + \arctan\!\left(\frac{\vec U_n\cdot \hat n_L^{(0)}}{\vec U_n\cdot\hat n_D^{(0)}}\right)$$

ここで $\hat n_D^{(0)}, \hat n_L^{(0)}$ はケーブル断面の固有座標系．

実装は風洞試験データを **[α, C_D, C_L]** のテーブルとして読み込み，線形補間：

```cpp
struct AeroPolar {
   std::vector<double> alpha_table;  // [rad]
   std::vector<double> Cd_table;
   std::vector<double> Cl_table;
   double Cd(double a) const;        // linear interp + clamp
   double Cl(double a) const;
};

// networkLine に：
const AeroPolar* polar = nullptr;  // 共有ポインタで OK
```

円柱の場合は $C_D \approx 1.2$ 一定，$C_L = 0$．D 型・矩形では
**Den Hartog 条件**[⁵][⁶]

$$H(\alpha_0) = \frac{\partial C_L}{\partial\alpha}\bigg|_{\alpha_0} + C_D(\alpha_0) < 0$$

を満たす迎角域があれば，それが**ギャロッピング不安定**を生む．**準定常モデルが
自動的に再現する**のがポイントで，特別な実装は不要．

#### ギャロッピング臨界風速

線形理論から[⁵]：

$$U_\text{cr} = \frac{4\,m_n\,\omega_n\,\zeta_s}{\rho_\text{air}\,D\,|H(\alpha_0)|}
= \frac{2\,Sc\,\omega_n D}{|H(\alpha_0)|}$$

検証では $U > U_\text{cr}$ で振動が指数的に発散することを確認する．

#### 検証

1. **円柱断面**：$H = C_D > 0$ なのでギャロッピング発生せず
2. **D 型断面**（$C_L'=-3.0, C_D=1.0$ 程度）：$H < 0$ で発散．
   臨界風速が上式と一致するか
3. **対数減衰率の風速依存性**：構造減衰 + 空力減衰 + ギャロッピングが
   重ね合わさった結果が $\zeta_\text{tot}(U)$ と整合するか

### Level 3 — Davenport 乱流入力（バフェッティング）

**目的**：風の時間変動による広帯域加振．モード重ね合わせで複数モードが同時励振される．

#### 物理

風速を**平均成分＋乱流成分**に分解：

$$\vec U_\text{wind}(\vec X, t) = \bar U(\vec X)\,\hat x + \vec u'(\vec X, t)$$

3成分乱流のうち，橋梁工学で支配的なのは**主流方向 $u'$ と鉛直 $w'$**．
ケーブルでは更に簡略化して $u'$ のみ考えても可．

#### Davenport スペクトル

最も古典的かつ広く使われる主流方向 PSD[⁷]：

$$S_u(f) = \frac{4\,K\,U_{10}^2\,x^2}{f\,(1+x^2)^{4/3}},
\qquad x = \frac{1200\,f}{U_{10}}$$

- $U_{10}$: 高度 10 m の平均風速
- $K$: 地表粗度係数（開けた地形 $\sim 0.005$，市街地 $\sim 0.05$）

最近では **Kaimal**, **von Karman** スペクトルも標準的だが，Davenport は形式が単純で
最初の検証ケースに向く．

#### 多点間コヒーレンス

橋梁ケーブルは長いので，**離れた2点の乱流は完全相関でも完全独立でもない**．
**Davenport コヒーレンス関数**：

$$\mathrm{Coh}(\Delta\vec X, f)
= \exp\!\left(-\frac{f\sqrt{C_y^2\Delta y^2 + C_z^2\Delta z^2}}{\bar U_m}\right)$$

経験定数 $C_y \approx 16, C_z \approx 10$．$\bar U_m$ は2点の平均風速の平均．

#### 時系列合成：Shinozuka 法

ケーブルを $N_p$ 点でサンプルし，多変量 Gauss 過程として時系列を生成．**N 点 × 時刻
ステップの2次元配列**を事前に生成しておき，シミュレーションでは補間で取り出す[⁴]．

$$u'_j(t) = \sqrt{2}\sum_{m=1}^{j}\sum_{l=1}^{N_f}|H_{jm}(\omega_l)|\sqrt{2\Delta\omega}
\cos(\omega_l t - \theta_{jm}(\omega_l) + \phi_{ml})$$

- $\mathbf{H}(\omega)$: クロススペクトル密度行列の Cholesky 分解
- $\phi_{ml}$: 一様乱数位相
- $\theta_{jm}$: $\mathbf{H}$ の偏角

数式は重そうに見えるが，**事前計算で配列に落とせる**ので実行時コストは線形補間だけ．

#### 必要なクラス

```cpp
class TurbulenceField {
public:
   // 構築時に Shinozuka 法で時系列を全点生成
   TurbulenceField(const std::vector<Tddd>& sample_points,
                   double T_total, double dt_sample,
                   double U_ref, double K_roughness);

   // 補間して取り出す
   Tddd uPrime(const Tddd& X, double t) const;

private:
   std::vector<Tddd> points_;
   std::vector<std::vector<Tddd>> u_history_;  // [point][time]
   double dt_sample_;
};

// 使用：
auto turb = std::make_unique<TurbulenceField>(...);
bridge_cable->wind_field = [&](Tddd X, double t) {
   return Tddd{U_mean, 0., 0.} + turb->uPrime(X, t);
};
```

#### コスト見積

- メモリ：$N_p \times N_t \times 3$ doubles
  例：$N_p = 50, N_t = 10^5, 3$ → 120 MB．許容範囲
- 構築：$N_p^2 \times N_f$ FFT．数秒〜数十秒
- 実行時：補間のみ（数 ns/呼出）．無視できる

#### 検証

1. **PSD の一致**：生成した時系列を FFT して入力 $S_u(f)$ を再現するか
2. **コヒーレンスの一致**：2点 $u'$ のクロススペクトルが入力コヒーレンス関数と一致するか
3. **モード重ね合わせ応答**：複数の構造モードが同時励振され，
   モード分解後の RMS 振幅が周波数領域 buffeting 解析（モード毎に
   $\sigma_n^2 = \int |H_n(\omega)|^2 S_F^{(n)}(\omega) d\omega$）と一致するか

### Level 4 — 結合 wake oscillator（VIV ロックイン）

**目的**：渦放出と構造振動の双方向結合を表現．ロックイン現象の再現．

#### 物理

各 `networkLine` に**無次元 wake 変数 $q(s,t)$** を持たせ，
van der Pol 型方程式で発展させる．Facchinetti–de Langre–Biolley (2004)[⁸]：

$$\ddot q + \varepsilon\,\omega_s\,(q^2 - 1)\,\dot q + \omega_s^2\,q = A\,\frac{\ddot u_n(s,t)}{D}$$

ここで

- $\omega_s = 2\pi f_s = 2\pi St\,U_n/D$：Strouhal 角周波数
- $u_n$：ケーブルの法線方向変位（$\hat n_L$ 方向成分）
- 経験的定数：$\varepsilon \approx 0.3$，$A \approx 12$，$\widetilde{C}_{L0} \approx 0.3$

揚力は

$$\vec f_\text{lift} = \tfrac{1}{4}\rho_\text{air}\,U_n^2\,D\,\widetilde{C}_{L0}\,q\,\hat n_L$$

q が van der Pol のリミットサイクル（振幅 2）に乗るので，
平均揚力係数は $\widetilde{C}_{L0}/2 \approx 0.15$ 程度になる．

ロックインの仕組み：
- 構造が振動 → 加速度項 $A\ddot u_n/D$ が q を駆動
- q の周波数が構造振動数に引きずられる（同期化）
- 結果として $f_s$ が固定値 $St\,U_n/D$ から外れて構造振動数に追従

これは **Level 1 の単純強制モデルでは絶対に出ない現象**で，業界標準モデルとして
40 年以上使われてきた[⁸]．

#### 必要な追加メンバ

```cpp
// networkLine に
double wake_q     = 0.0;   // q
double wake_qdot  = 0.0;   // dq/dt
// 経験定数（クラス共通でもよい）
double wake_eps   = 0.3;
double wake_A     = 12.0;
```

#### 時間発展：既存 RK サブステップに乗せる

`networkLine` の wake 変数を**節点と一緒に同じ RK4 で進める**のが最も整合的．

`MooringLine::simulate()` のサブステップループ内で（既存の節点更新と同じ場所）：

```cpp
// 各セグメント：q の RK4 段
for (auto& l : this->getLines()) {
   // u_n とその2階微分を法線方向成分から取り出す
   // u_n = (X_l - X_l^equilibrium) · n_L
   // ddot u_n は a_n を n_L に射影して得る
   auto a_n = (a_p1 + a_p2) * 0.5;     // 隣接2節点の加速度平均
   double accel_n = Dot(a_n, n_L);

   double omega_s = 2*M_PI * l->St * Un / l->diameter;
   double q  = l->wake_q;
   double qd = l->wake_qdot;
   double qdd = -l->wake_eps * omega_s * (q*q - 1) * qd
                - omega_s*omega_s * q
                + l->wake_A * accel_n / l->diameter;

   // RK4 段は MooringLine::simulate() のループに合わせて push
   l->RK_q.push(qd);
   l->RK_qdot.push(qdd);
}
```

ここで `RK_q`, `RK_qdot` は `networkLine` に追加する `RungeKutta<double>` メンバ．
**節点と同じ `dt_cfl_stage` で進めれば連立 ODE として整合**する．

#### CFL 追加制約

$\omega_s$ は風速で決まるので，明示積分の安定条件に

$$\Delta t \lesssim \frac{C_\text{cfl}}{\omega_s}
= \frac{C_\text{cfl}\,D}{2\pi\,St\,U_n}$$

を追加．例：$U_n = 30$ m/s, $D = 0.15$ m → $\omega_s \approx 250$ rad/s →
$\Delta t \lesssim 0.4$ ms （$C_\text{cfl}=0.1$）．既存の弦 CFL より厳しい場合があるので，
`MooringLine::get_dt()` に wake 用の項を追加：

```cpp
double dt_wake = C_VW * l->diameter / (2*M_PI * l->St * Un);
if (dt_cfl > dt_wake) dt_cfl = dt_wake;
```

#### 初期条件

q の初期値は小さな乱数（$\sim 0.01$）にして，リミットサイクルに自然に乗せる．
ゼロから始めると数値ノイズで不安定になることがある．

#### 検証

1. **静止構造**：$\ddot u_n = 0$ で q が振幅 2 のリミットサイクルに収束するか
2. **強制振動下のロックイン**：構造1次モード周波数 $f_n$ を固定し，風速をスイープ．
   $f_s/f_n \in [0.85, 1.15]$ 付近で q の周波数が $f_n$ に同期するか
3. **Skop–Griffin プロット**：[$Sc$, 振幅] 平面でデータを取り，Skop–Griffin の
   経験曲線と比較

## 5. 推奨実装順

**フェーズA（最小実装，1〜2週）**：
- L0（定常抗力）→ L1（強制揚力）まで．
- 検証：弦理論固有振動数，静的たわみ，VIV 弱応答．

**フェーズB（実用版，2〜4週）**：
- L2 を追加．迎角依存の空力係数テーブル読込．
- 検証：ギャロッピング臨界風速の Den Hartog 公式照合．

**フェーズC（リサーチ版，1〜2ヶ月）**：
- L3（Davenport 乱流）と L4（wake oscillator）を独立に追加．
- L3 と L4 は競合せず**両方使える**：乱流入力で広帯域加振しつつ，VIV 周波数帯で
  ロックインも再現する．
- 検証：実橋計測との比較，DeepCwind 風試験データ等．

## 6. 共通の注意事項

### (i) 風速場関数の渡し方

`std::function<Tddd(Tddd, double)>` をクラスメンバにして，setter で差し替え可能に：

```cpp
bridge_cable->wind_field = [](Tddd X, double t) {
   double U10 = 30., z = X[2], alpha = 0.16;
   return Tddd{U10 * std::pow(z/10., alpha), 0., 0.};
};
```

これで「一様風」「べき乗則」「乱流入力」を**実装変更なしに切替**できる．

### (ii) 既存 `getDragForce()` との共存

海中係留 cable と橋梁 cable を同じ実行ファイル内で扱う場合，
**派生クラス `BridgeCable : public MooringLine`** を作って `getForce()` を
オーバーライドするのがクリーン．既存の `MooringLine::simulate()` の
力評価ループ部分を仮想関数化する小さなリファクタが必要になるが，
それ以外は変更不要．

### (iii) 出力の整備

橋梁特有の出力項目：
- 節点変位の時刻歴 → モード分解（FFT）
- 動的張力増分 $\Delta T(t)$
- セグメント揚力 / 抗力の時刻歴
- 実効減衰（自由振動の包絡から逆算）
- L3 では入力風速の PSD と出力応答の PSD の両方
- L4 では q の時刻歴と Lissajous 図（ロックイン同期の可視化）

### (iv) パラメータ管理

風荷重パラメータが増えるので，JSON 入力ファイル化が望ましい：
